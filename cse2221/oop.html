Building Blocks of OOP

In Java there are two very important concepts; Classes and Objects

Classes

That bit of code near the top of every Java file that we’ve been writing up until now, public class, has actually been foreshadowing this part of the course. Every file that you’ve written has actually been a class, but what is a class?

A class is a framework that an object follows. They have methods that are both declared and implemented and objects can be instantiated from them. These objects will have all of the methods from the class and they type of the object will be the name of the class it is.

Classes can be thought of as a mold. This mold can be used to create objects. When an Object is created from a Class it is reffered to as a Child of the class.

  • Classes can define certain properties of the Objects created from them. They can also leave properties for their children to define themselves later.
  • New classes can also be created based on another class, this process is called Inheritance

Objects

An object is simply an instantiated class. We have been working with objects all year, the code 

NaturalNumber n = new NaturalNumber2();

Creates an object n that instantiates the class NaturalNumber.

  • An object can be referenced inside of its class with the this keyword.

Objects are essentially boxes that hold methods and properties. These can be called upon, changed or accessed like any data but they are associated with the object that contains them.

Interfaces

And Interface is similar to a class but instead of implementing methods, it only declares them. They are made to define a loose framework for a class to follow. Any class that inherits from an interface has to implement all of the methods defined by the interface as well as any other methods it needs that are not declared by the interface.

Inheritance

Inheritance is a central idea of Object-oriented programming. It is the idea that Classes can extend other classes and implement interfaces.
