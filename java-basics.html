What is Java

Java is a fairly popular object-oriented programming language.

  • Java was programmed in the 1990’s by Sun Microsystems.
  • It was designed to be an easier to understand object-oriented programming language.
  • Java Applets used to be a thing, why are we stil talking about them?

Compilation

The Java Compiler runs thorugh the source code of a .java file and generates bytcode from it into a .class file.

  • Eclipse using it’s own compilier that continuously compiles code as it’s being written.
  • javac is the compilation command included in the JDK

Java Virtual Machine

The Java Virtual Machine is essentially a virtual computer.

  • The JVM is like any other program that runs on a physical chip.
  • When you run a java (.class) file, your computer creates an instance of this virtual computer and runs your code on it.
  • The JVM only runs bytecode generated from a Java file.

The JVM is super significant the the Java progamming language because it follows the Java slogan of “Write once, run anywhere.”

  • The JVM is also universal, allowing other programming languages like Clojure to compile to the JVM.
  • The JVM is also very performance friendly, the extra layer it adds to Java does not cost a lot of resources.

SimpleWriter

When creating an instance of a SimpleWriter1L, if you pass a file name then all output will be sent to that file. However, when left blank, the default behavior of console output is used.

Simple Reader

SimpleReader is very similar to SimpleWritter in how it is created. However, instead of handling output, SimpleReader handles input. 

SimpleReader keyboardIn = new SimpleReader1L();
SimpleReader fileIn = new SimpleReader1L("foo.txt");

String line = keyboardin.nextLine();
line = fileIn.nextLine();

The .nextLine() method reads everything up until a line-separator or the end of console input.

Just like with SimpleWritter, SimpleReader must be closed at the end of the program with the .close() method.

In Java there are two very imporant concepts; Classes and Objects

Classes can be thought of as a mold. This mold can be used to create objects. When an Object is created from a Class it is reffered to as a Child of the class.

It is important to draw the distinction between program variables and mathematical variables because despite both being variables they function in discretely different ways.

  • A program variable has a particular value at any given time during program execution, and that value may change.
  • However a mathematical variable stands for an arbitrary but fixed value.

  • A program variable has a corresponding mathematical type that models it.

Parameters

Inside of the parenthesis of a method declaration are where parameters are defined. Parameters essentially define the information the needs to be provided to the function in order to run. They are incredibly useful for getting values to a method that would otherwise not have access to them due to scope.

  • A demonstration running through this idea with an example function can be found in the lecture slides.
  • An important thing about parameters is that they are not actually the variable passed in but instead references, which are essentially copies of the value of the variables.
  • Parameters only exist in their parent function, they cease to exist when the function ends.

An array is a group of similar variables all of the same type. They are refferenced by the name of the array, followed by the index of the value, ex myArray[0]. The Index of an array is always a positive integer and reffers to the value’s place in the array.

Static Methods

A Static method is a block of code with a name. This allows it to be easily run again my simply calling the method name in the format methodName(). The method essentially takes over comptutation when it is called until it returns where computation resumes after where the method was called.

Anatomy of a Static Method

private static int distance(int a, int b) {
    int c = a - b;
    return c;
}

The first line of a method is its declaration.

  • The private static tells us that the method is static
  • int is the return type of the method meaning that the method will return something with the type int.
  • distance is the name of the method, it’s what’s used to call this method.
  • (int a, int b) specifies that this method takes two parameters of the type int. Parameters are values that are passed to the method when it’s called.

Everything inside of the two curly braces ({}) is referred to as the Method Body. This is the code that is executed when the method is called.

  • int c = a - b is a line that uses the parameters in a calculation. Notice how the parameters don’t have to be initialized.
    • int c is a local variable, meaning that it only exists inside the method and cannot be accessed anywhere else.
  • return c means that this method will send whatever the value of c is to where it was called. Since this method is of type int, the value it returns must be an int.

Also most methods are named using camelCase. This format starts with a lower-case word and each subsequent word is added to the end of the name (with no spaces) with the first letter capital.

Important Rules

  • Do not use break to end loops
  • Use only one return in functions
  • Do not use return in procedures
  • Try to remove all warnings (compilier, Checkstyle, Spotbugs) from your program in particular:
    • Follow the naming conventions
    • Declare named constants for every literal that is a magic number
    • Do not assign new values to formal parameters
