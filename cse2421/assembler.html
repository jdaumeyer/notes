Why Assembler

Performance

Assembler is the closest that we can get to actual machine code without writing binary.

Differences From C

  • Where C handled byte math for us, Assembler offers no such luxury. You will have to know the exact amount of bytes that you need for everything.

Storage

Registers

Whenever you do math in Assembler you want to be using Registers, because they are the fastest storage that we have available.

  • The machine that stdlinux uses has 16 registers in total, however 2 are already reserved so we only realistically have 14 to use.
  • Each register has some purpose associated with it, and some registers are especially efficient at specific tasks then others.
  • All registers can store a quad or 8 bytes.

Opcodes

ALU Flags

Operations that run through the ALU (so all math operations) can set flags. In x86 we can set 4 different flags:

ZF
“Zero Flag”, Set if the last operation in the ALU resulted in a 0.
SF
“Sign Flag”, Set if the last operation in the ALU had a 1 as the most significant bit.
OF
“Overflow Flag”, Set if the last operation in the ALU overflowed.

Directives

Just like with C, Assembler also has directives that inform the pre-processor about the program. However instead of starting with a #, Assembler Directives start with a ..

.file   required

.file specifies the name of the source code file. This has to be set in order for the linker to work properly. 

.file "filename.s"

.section   required

.section splits the assembly code into different sections.

.rodata

The following data will be placed in the read only portion of memory. 

.section .rodata

.text

Changes the section to text/code 

.text

.data

Changes the current section to the data section 

.data
Array:
    .quad 0x6f
    .quad 0x84

.global   required

  • Always followed by the name of a function

  • Needed by the linker for symbol resolution 

    .globl main

.type   required

  • Needed by the linker to know that an address points to a function, not data 

    .globl main
    .type main, @function

.size   required

  • Used by the linker to know the size needed for the program.

.string

  • Stores the data as a null terminated string in memory. 

    .string "Hello, world!"

.quad

Places the given value into memory encoded in 8 bytes

.long

Places the given value into memory encoded in 4 bytes

.word

Places the given value into memory encoded in 2 bytes

.byte

Places the given value into memory encoded in 1 byte
