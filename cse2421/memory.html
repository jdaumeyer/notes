Memory

“Memory is a sea of byte addressable bits.” — Neil Kirby

What is Memory

Memory is just 1’s and 0’s, what it stores is determined by how we read it. For integers alone there are multiple ways we can read the same bits.

Bits Simple Integer Two’s Complement
0000 0 0
1010 10 -6
0110 6 6

We can chose which of these interpretations that C will use with the keyword unsigned. By default C uses Two’s Complement but with the keyword unsigned it will use simple integer interpretation.

Pointers

A pointer is an 8-byte address to a location in memory.

Pointer Operators

There are two ways that we can obtain a pointer.

&
The address operator gets the address of the provided variable. So &x would be a pointer to x
*
The dereferrence operator is how we access the data of a pointer and also how we assign a pointer variable.
  • int *y is an int whereas y would be the pointer.

Types of Pointers

Void Pointers

A void pointer is the most basic type of pointer. It points to some address of memory but doesn’t assign it a type. Because of this void pointers cannot be dereferenced, as the language doesn’t know how to read the data that they’re pointing to.

  • If we want to assign a type to a void pointer, we have to cast it.

Data Pointers

A data pointer implies a type to the data it’s pointing at. 

int *x;
double *y;
  • x is a pointer to int, and *x is an int.
  • y is a pointer to double and *y is a double.

Function Pointers

Function pointers allow us to treat behavior as data. They allow us to pass a function as a parameter to another function.

  • Just like arrays where the name of the array is a pointer to the first element, the name of a function is a pointer to that function.

Why Pointers

Pointers allow us to change data in a function. This is otherwise impossible because C is pass by value.

  • We are actually still passing a copy of the pointer. But we can still use it to change what it’s pointing to.
  • With function pointers, we can have a function call the one that we pass.

Static and Dynamic

Memory is typically seen as a dichotomy between Static (Known at compile time) and Dynamic (Memory that’s allocated while the program runs).
