Chapter 1

Computer system structure

Divided into 4 components:

  • Hardware
  • OS
  • Application Program
  • People, machines and other computers

No universally accepted definition for an operating system The program that runs at all times is called the Kernel

  • The first entry for ps -ef

Everything else is a

  • System Program
  • Application Program
  • Middleware is used for distributed environments

An interrupt is an alert that a controller has finished its task

  • Trap = Exception

Multiprogramming

  • A single user cannot keep CPU and I/O devices busy at all times
  • One job selected and run at a time using job scheduling

Multitasking

Swapping between process as fast as possible to make it appear that multiple things are happening at once

  • Now we can actually multitask more effectively with multiple core processors

Dual Mode

Allows us to distinguish between Kernel code and user code.

Kernel Mode
Handles system calls and resource management
User Mode
Runs user processes

Process Management

A process is a program in execution

Process Management Activities

  • Create and Delete Processes
  • Suspending and Resuming
  • Synchronization
  • Communication
  • Deadlock Handling

Memory Management

All or part of a program must be in memory before execution.

  • Memory Management Controls What is in memory and when

File System Management

Files are simply abstract data and it’s up to the OS to determine how to handle it.

  • Creating and deleting files
  • Manipulating files primatively
  • Mapping files to a secondary drive
  • Controlling who has access to files

Protection and Security

Protection
is and mechanism for controlling access
Security
is any defense from attacks
