Scheduling Criteria

CPU Utilization ⬆
Keep the CPU busy
Throughput ⬆
# of processes that go through the CPU
Turnaround Time
The time it takes to fully complete a process (from submission to termination)
Waiting Time ⬇
The amount of time that a process spends in the ready queue
Response Time ⬇
The time it takes for a process to start responding to the user
  • ⬇ want to minimize, ⬆ want to maximize

Scheduling Algorithms

These will for sure be on the midterm

Gantt Chart Problems

First Come First Serve

Shortest Job First

  • Preemptive vs. Non-preemptive
    Preemptive
    Running shorter run time processes as they come in
    Non-preemptive
    Finish the current process before looking for the next

Round Robin (RR)

Yum!

Processes are cycled onto the CPU to run for time \(q\) before being moved to the back of the ready queue.

  • We end up doing more time context switching rather than actually executing processes making this inefficient

Priority Scheduling

Processes are ranked on their priority (lower numbers typically mean high priority). The CPU will then execute the process with the highest priority.

  • Can lead to Starvation where a process never runs because there is always a new higher priority process
    • Fixed with Aging where priority slowly increases while to process is not getting execution time
  • Non-Uniform Memory Access
