Tree Review

Trees are an undirected (meaing there is no specific direction to follow) graph structure.

  • A Rooted Tree is a tree with a designated Root Node
  • An Ordered Tree cares about the order of its children.

Binary Tree

A binary tree is a rooted tree where every node has at most two children. Because of this, every binary tree will have exactly two subtrees, but one or both might be an empty tree.

Tree Traversals

A Tree Traversal is a path that visits each node of a tree once.

Because we can’t access BinaryTree directly, there is no way to do a breadth first traversal. We can only use Depth First Traversals

Depth First Traversal
This is a traversal method that follows each path to its end
Breadth First Traversal
This is a traversal method that looks at all the immediate nodes and moves from there

Ordered Traversals

It can be important to know which node we are accessing first. We describe this as an ordered traversal.

Preorder Traversal
The Root node is accessed first (Root \(\rightarrow\) Left \(\rightarrow\) Right)
Inorder Traversal
The Root node is accessed after the left node (Left \(\rightarrow\) Root \(\rightarrow\) Right)
Postorder Traversal
The root is accessed last. (Left \(\rightarrow\) Right \(\rightarrow\) Root)

This is something you should be expected to know on an exam.

Binary Search

A binary search is conducted by looking at the middle element or an already sorted set and moving depending on if the middle element is greater than or less than the element we’re looking for.

Binary Search Tree

A Binary Search Tree (BST) is a special kind of tree where all of the nodes are ordered in a Mathematical Preorder (different than a Preorder Traversal). They are super useful because they allow us to find items in logarithmic time.

  • A minimal element is the leftmost entry and a maximal element is the rightmost node.
  • The empty tree is a valid Binary Search Tree. So when programming you have to account for the empty subtree.

Item Equivalency

Because there are some elements that can have an equivalent order but be different elements, it’s important to compare using the root node first! 

root.compareTo(child)
compare(root, child)
  • A positive number means they’re out of order.
  • A negative number or zero means they’re in order.
