Hashing

Hashing is one of these techniques to reduce time. It’s main goal is to reduce the amount of time to perform a data search. It works by using a Hashing Function to sort data.

The Hash Function

A Hash function is part of what a hash does. It is a function that in a piece of data and returns a numeric value based on it.

  • The .hashCode() function in Java returns an int value.
  • Hash functions have to be deterministic meaning that for every equal value, it must return the same thing.

The Process

  1. run the Hash Function on an object.
  2. The Hash function returns a Hash Value.
  3. Fit the Hash Value to the length of your hash table to get the index.
    • This is typically done with modulus hash_value % length
  4. use the bucket at that index.

Mod v. Modulus

The % operator in java is not a “Modulus” operator. Instead it returns the remainder.

  • The difference occurs at negative numbers, if the remainder of something is negative add the number back in.

      -4 -3 -2 -1 0 1 2 3 4
    n / 3 0 -1 0 0 0 0 0 1 0
    n % 3 -1 0 -2 -1 0 1 2 0 1
    n mod 3 2 0 1 2 0 1 2 0 1

Drawback and Benefits

Hashing is only going to be fast if the Hash Function is fast as well. Because of this function, there will always be a small performance hit but it shouldn’t be a major one.
