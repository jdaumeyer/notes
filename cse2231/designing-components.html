Object Oriented Programming

Abstract Classes

An Abstract Class is what bridges the gap between an Interface and a Class. They can’t have objects instantiated from them and are allowed to have abstract methods.

Kernel Class

Kernel Classes aren’t really a thing outside of Ohio State. But a Kernel Class is essentially just a class that has a data representation.

  • They have a data representation that can be manipulated
  • There are rules on how the data represents the object value
  • There are rules on how the data is stored
  • The Kernel only represents the Abstract

We want to keep the Kernel class as small as possible and implement as much as we can in the Abstract class.

  • Because everything depends on the Kernel class, they can be hard to debug.

Template Method is a design pattern for dealing with different implementations of the same thing.

The Object Class

Object is a super class of every concrete type. The object class defines three methods that all objects will have:

  • toString()
  • equals()
  • hasCode()

Systems

Systems are a hard thing to define, but for our needs a system is just some implementation. Every system is made up of smaller subsystems.

You can think about a system in two different ways:

Abstract View

This is what the system does as a whole. It’s the outside view of how the system interacts with other systems. The Abstract view is the view of what a system does.

Concrete View

This is how the pieces of the system interact. And the view of the system from inside itself. The Concrete View is the view of how a system works.

  • When using a Concrete View of a system, it is necessary to have and Abstract View of its pieces as to not overcomplicate things.

When working with large software we want to break concrete-to-concrete dependencies. Because with a concrete-to-concrete dependency when we change code of one, we could break the other. Object Oriented Programming and Design By Contract are both methodologies designed to prevent concrete-to-concrete dependencies.

State Spaces

Abstract State Space

The Abstract State Space of a type is all the values that it can take on, from the client’s perspective.

  • The Abstract State Space for NaturalNumber is all non-negative numbers.

Concrete State Space

The Concrete State Space is the set of all possible values that a specific data representation can have.

  • This is what a class can actually represent, not what it’s modeling

Abstract Function

The Abstract Function or the Correspondence function, is a function that turns the Concrete State Space into the Abstract State Space.

Kernel Purity Rule

The Kernel Purity Rule (KPR) states that no method in a component with a data representation may never call a public method of itself

  • This is used to cut down on those concrete dependencies that we want to try our hardest to avoid.
  • If you need to do recursion, do it in a private method.
