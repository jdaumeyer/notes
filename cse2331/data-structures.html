Hash Tables

Uses a Hash Function to generate keys (or locations) of where to put values in an array. There are two different methods for dealing with duplicate keys:

Open Address Hashing
Generates a new key when there is a collision
Chained Hashing
Appends duplicate keys to a linked list at that location

There are two important notes about our implementation:

  • Keys have to be immutable
  • Duplicate keys do not work

Problems

  • There are “bad” hash functions that can result in a poorly performing hash table. A “good” hash function should return as unique as a value for every input as possible.
  • Hash Tables are really fast for storing data (i.e. inserting and finding elements) however they are bad at things like finding the max or min.

Binary Search Tree

A tree where all children less than the parent are in the left subtree and all children greater are in the right subtree.

  • Good at intervals and generic searches

Problems

  • The order of elements can effect the performance of the tree. If they’re all inserted in decreasing order then the tree will effectively become a linked list.
  • The rotate methods require parts of the tree to not be null, however this isn’t guaranteed.

Red Black Tree

A self-balancing tree.

  • Whenever you work with a binary tree in code it’ll likely be some form of self-balancing tree underneath.

Problems

The coloring of nodes is very important to a Red Black Tree. Because of this not all Binary Trees are Red Black Trees however all Binary Trees can be turned into Red Black Trees through rotations.
