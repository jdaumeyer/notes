Course Introduction

Analog Vs. Digital

In this class we will mainly deal with Digital Signals but it’s important to note the differences between the two

Analog
Can have any value over continuous time
  • All “real” signals are Analog
Digital
Must have a value that can be represented by the number of binary bits, measured over discrete time.
  • Use logic values instead of voltages
  • Used for data processing and communications

Logic Circuits

Combinational Logic
Output only depends on inputs
Sequential Logic
Output depends on current and past inputs (memory)
Synchronous Logic
Functions are timed with a clock
Asynchronous Logic
No clock is present

Number Systems

Numbers can have different bases (or radix). Isn’t that cool.

\begin{align*} 469.14_{10} = 4 \cdot 10^2 + 6 \cdot 10^1 + 9 \cdot 10^0 + 1 \cdot 10^{-1} + 4 \cdot 10^{-2} \end{align*}
  • A decimal point is called a radix point for any non base 10 number

Conversion Between Bases

To convert a number from base \(B_1\) to base \(B_2\)

  • For all digits before the radix point, we divide by the new base (\(B_2\)) until we get to 0. The remainder of each division are the digits for the number in \(B_2\).
  • For all digits after the radix point, we multiply by the new base (\(B_2\)). However because fractions don’t translate well into new bases, we only need to do this for the precision we want. The digit for our new number is the value before the radix point after multiplication.

Binary Arithmetic

Arithmetic in digital systems is always done in binary for simplicity and efficiency. Because this is a digital logic course, we look at how these operations would actually be done on a computer.

Multiplication

Table 3: Binary Multiplication Table
\(0_2 * 0_2\) \(0_2\)
\(0_2 * 1_2\) \(0_2\)
\(1_2 * 0_2\) \(0_2\)
\(1_2 * 1_2\) \(1_2\)

We do binary multiplication by finding all our partial products and then adding them together. However it can be easier and more accurate to add the partial products as we go and not all at the end.
