The terms Software Development Process(SDP) and Software Development LifeCycle(SLDC) are used interchangeably

Processes

Process
A sequence of steps that define some operation which add cost but reduce risk

Processes are repeatable, predictable, traceable, improvable and trainable.

Designing a Process

To first define a process for software engineering we must first understand software engineering. The key aspects to know are the stages and phases.

Phases

Phases
tasks grouped by common intent, that are not in an order

Phases of software development include:

Requirement Identification
identifying the problem to solve
Analysis
understanding the domain, problem and solution
Architecture and Design
drawing out the underlying structure
Implementation
actually building the system
Testing
checking that the system meets requirements
Deployment
packaging and releasing the system for user access
Maintenance
keep the system working and useful
Project Management

Stages

Stages of software development

  1. Inception creating a vision and generating estimates
  2. Elaboration refine the vision and validate architecture
  3. Construction development and preparation for deployment
  4. Transition Deployment

Current Processes

There are two well defined processes for software development:

Waterfall / Plan Driven

A sequential process where each phase has to be finished before the next one can begin

  • Good with a well defined scope or project. i.e. requirements wont change much during development

Agile / Iterative

An iterative process where every new batch of features undergoes its own phases of development before the next feature(s) are worked on.
